      📖 À propos de MediaGrab (lisez-moi)

      Ce que MediaGrab peut faire

      • Télécharger un lien direct vers un média (avec barre de progression réelle quand le serveur envoie Content-Length).
      • Importer des fichiers locaux (glisser-déposer ou sélecteur), les prévisualiser et lire leurs métadonnées.
      • Convertir une image côté client via <canvas> (PNG ↔ JPG ↔ WebP).
      • Gérer une file d'attente de plusieurs URL, et un historique persistant.
      • Basculer entre thème sombre / clair (préférence mémorisée).

      Ce que MediaGrab ne peut pas faire

      Impossible de télécharger depuis YouTube, TikTok, Instagram, LinkedIn, Facebook, X/Twitter. Ces plateformes servent leurs médias via des flux protégés et bloquent l'accès inter-origines (CORS). Un vrai serveur backend (type yt-dlp) est alors indispensable. MediaGrab est volontairement honnête : il détecte ces liens et l'explique, plutôt que de simuler un faux téléchargement.

      Pourquoi certaines URL directes échouent-elles quand même ?

      Même un lien .mp4 valide peut être refusé si le serveur distant n'autorise pas le CORS. Dans ce cas, MediaGrab vous propose une solution de repli : ouvrir le média dans un nouvel onglet et faire clic droit → Enregistrer sous.

      Architecture : 3 fichiers — index.html (structure), style.css (thème sombre/clair, responsive), app.js (logique). Aucune dépendance, aucun CDN, aucun build. Ouvrez simplement index.html.
